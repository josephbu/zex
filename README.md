# ZEX — Zip Export EXpress

Convert a ZIP of Canva slide exports (numbered PNGs) into a downloadable **PDF** and **PowerPoint (PPTX)** file — directly in the browser, via a single PHP file dropped on any shared web host.

No Node.js. No composer required. No external services. Just PHP.

---

## What it does

Canva lets you export a presentation as individual PNG images, zipped up and numbered in order (`1.PNG`, `2.PNG`, etc.). ZEX takes that ZIP and turns it into:

- **PDF** — full-bleed widescreen, one slide per page
- **PPTX** — valid Open XML PowerPoint file, each PNG filling a 16:9 slide at 1920×1080 equivalent

Both files are named after the original ZIP upload and offered as instant downloads. Temporary files are cleaned up automatically.

---

## Features

- Single-file PHP app — one `index.php`, nothing else required
- Drag-and-drop or click-to-browse ZIP upload
- Natural sort ordering — `10.PNG` comes after `9.PNG`, not `1.PNG`
- macOS metadata files (`._filename`, `__MACOSX/`) automatically skipped
- Upload progress bar with live byte counter
- Output filenames match the original ZIP name, spaces and all
- Automatic temp file cleanup (session folders older than 1 hour deleted on each request)
- Three built-in colour themes, switchable via a single config line
- No database, no login, no dependencies

---

## Requirements

- PHP 7.4+ (PHP 8.x recommended)
- Extensions: `zip`, `gd` — both enabled by default on most shared hosts
- Write permission on the `tmp/` directory

Check your extensions are available:

```bash
php -m | grep -E "zip|gd"
```

## Installation

```bash
git clone https://github.com/josephbu/zex.git
cd zex
mkdir tmp && chmod 775 tmp
```

Upload `index.php` and the `tmp/` directory to your web host. Open the URL in a browser.

That's it.

## Configuration

All options are defined at the top of `index.php`:

```php
// ─── Config ──────────────────────────────────────────────────────────────────
define('MAX_MB',     100);       // Maximum ZIP upload size
define('PPTX_W_EMU', 9144000);  // Slide width  (EMU) — 10 inches, 16:9
define('PPTX_H_EMU', 5143500);  // Slide height (EMU) — 5.625 inches, 16:9

// ─── Appearance ──────────────────────────────────────────────────────────────
// THEME: 'light-breezy' | 'business-clean' | 'dark-moody'
define('UI_THEME', 'dark-moody');
```

### Themes

| Value | Description |
|---|---|
| `dark-moody` | Dark background, purple/violet accent, Instrument Serif heading |
| `light-breezy` | Soft blue-white gradient, sky blue accent, Playfair Display heading |
| `business-clean` | Off-white, near-black accent, Fraunces serif heading — minimal and professional |

### Slide dimensions

The defaults match Canva's standard widescreen export (1920×1080 / 16:9). If your design uses a different canvas size, adjust the EMU values:

```php
// Square (1:1)
define('PPTX_W_EMU', 6858000);
define('PPTX_H_EMU', 6858000);
```

1 inch = 914,400 EMU.

---

## Upload size limits

ZEX checks `MAX_MB` in the app, but your PHP server config imposes its own ceiling. On shared hosting you can often raise these via `.htaccess`:

```
php_value upload_max_filesize 100M
php_value post_max_size 110M
php_value memory_limit 256M
```

`memory_limit` matters because ZEX reads every PNG into memory before writing it into the ZIP. A 30-slide deck at ~1 MB per image uses ~30 MB of RAM during PPTX generation.

## Optional: higher-quality PDF output

By default ZEX uses PHP's built-in GD library to generate PDFs. For improved quality, install [TCPDF](https://tcpdf.org/) via Composer:

```bash
composer require tecnickcom/tcpdf
```

ZEX detects it automatically — no config change needed.

## How it works

1. You upload a `.zip` containing `1.PNG`, `2.PNG`, `3.PNG` … etc.
2. PHP extracts the ZIP to a unique session directory under `tmp/`.
3. PNGs are sorted using natural sort (`strnatcasecmp`) so numbered sequences order correctly.
4. A PDF is generated with each PNG as a full-bleed page.
5. A PPTX is generated by building a valid Open XML package directly — no third-party library required. The structure is validated against the OOXML spec.
6. Both files are linked for download, named after the original ZIP.
7. On the next page visit, any session folders older than 1 hour are deleted.
